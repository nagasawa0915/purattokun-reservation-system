📋 コンポーネント統合フロー

1

個別開発

各コンポーネントを独立して開発・テスト

2

司令塔登録

SystemCoordinatorにコンポーネントを登録

3

協調初期化

依存関係を考慮した順序初期化

4

統合完了

全コンポーネントが協調動作

💻 コンポーネント登録例

// 司令塔にコンポーネントを登録
coordinator.registerModule('TimelineModule', TimelineModule, {
    phase: 'ui-controllers',           // 初期化フェーズ
    priority: 20,                      // 優先度
    dependencies: [],                  // 依存関係
    domTargets: ['.timeline-panel']    // 占有するDOM要素
});

coordinator.registerModule('OutlinerModule', OutlinerModule, {
    phase: 'interaction',
    priority: 30,
    dependencies: ['TimelineModule'],  // TimelineModule後に初期化
    domTargets: ['.outliner-panel']
});

🔧 個別テスト vs 統合テスト

// 【個別テスト】- 単体で動作確認
http://localhost:8000/components/timeline/timeline-standalone.html
- 独立した環境で機能テスト
- モックデータでの動作確認
- UI/UXの細かい調整

// 【統合テスト】- システム全体での動作確認  
http://localhost:8000/index-coordinated.html
- 他コンポーネントとの連携テスト
- 実データでの統合動作確認
- パフォーマンス・競合問題の確認
